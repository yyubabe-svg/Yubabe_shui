## 一、服务器选购指南

### 推荐配置（火山引擎）

| 使用人数 | 配置 | 带宽 | 月费（参考） | 推荐度 |
|---------|------|------|-------------|--------|
| 5-10人 | 2核4G | 5Mbps | 50-80元 | ⭐⭐⭐ |
| 10-30人 | **4核8G** | **10Mbps** | **100-150元** | **⭐⭐⭐⭐⭐（推荐）** |
| 30人+ | 8核16G | 20Mbps | 250-400元 | ⭐⭐⭐⭐ |

### 购买入口
- 火山引擎官网：https://www.volcengine.com/
- 产品路径：**云服务器ECS** 或 **轻量应用服务器**
- **地域选择：华北2（北京）** ⚠️ 重要！和火山方舟API同区域，内网访问延迟更低

### 系统选择
- 操作系统：**Ubuntu 22.04 LTS 64位**
- 系统盘：60GB SSD 起步（如果文档多建议100GB+）

## 二、购买与初始化服务器

### 1. 购买后获取服务器信息
购买成功后，在火山引擎控制台获取：
- **公网IP地址**（类似：123.456.789.0）
- **root密码**（购买时设置，或通过控制台重置）

### 2. SSH连接服务器
在本地终端（Mac/Linux）或PowerShell（Windows）执行：
```bash
ssh root@你的服务器IP
# 输入密码登录
```

### 3. 首次登录基础配置（可选但推荐）
```bash
# 更新系统
apt update && apt upgrade -y

# 安装常用工具
apt install -y curl wget git nano vim

# 设置时区为上海
timedatectl set-timezone Asia/Shanghai
```

## 三、安全组配置

⚠️ **这一步很重要，不配置将无法访问！**

### 在火山引擎控制台配置防火墙规则：
进入云服务器实例详情 → 「安全组」→ 「配置规则」→ 「入方向规则」，添加：

| 协议 | 端口 | 源地址 | 说明 |
|------|------|--------|------|
| TCP | 22 | 0.0.0.0/0 | SSH远程登录 |
| TCP | 80 | 0.0.0.0/0 | HTTP网页访问 |
| TCP | 443 | 0.0.0.0/0 | HTTPS（配置域名后需要） |

**注意**：8000、5173等端口不需要对外开放，内部通过Docker网络通信。

## 四、上传代码到服务器

### 方式一：使用Git（推荐）
如果代码在GitHub/Gitee等仓库：
```bash
# 在服务器上
cd /opt
git clone 你的仓库地址 shushui-ai
cd shushui-ai
```

### 方式二：使用SCP从本地上传
在你**本地电脑**的终端执行（不是服务器上）：
```bash
# 将整个项目上传到服务器/opt目录
scp -r /Users/yuma/Documents/水利电力勘测/水利员智库AI/shushui-ai root@你的服务器IP:/opt/
```

上传完成后，SSH到服务器进入项目目录：
```bash
cd /opt/shushui-ai
ls -la  # 确认文件都在
```

## 五、一键部署

### 1. 修改生产环境配置
```bash
cd /opt/shushui-ai
nano .env.prod
```

需要修改的关键配置：
```bash
# 将 YOUR_SERVER_IP 替换为你的服务器公网IP
SERVER_PUBLIC_URL=http://你的服务器IP
ALIPAY_NOTIFY_URL=http://你的服务器IP/api/payment/alipay/notify
ALIPAY_RETURN_URL=http://你的服务器IP/api/payment/return

# 建议修改以下安全密钥（随便改一个长字符串即可）
SECRET_KEY=你自己的随机长字符串
ADMIN_TOKEN=你自己的管理员密码
```

修改完成后按 `Ctrl+O` 保存，`Ctrl+X` 退出。

### 2. 给部署脚本加执行权限
```bash
chmod +x deploy.sh
```

### 3. 执行一键部署
```bash
./deploy.sh
```

⏳ **首次构建需要5-10分钟**（主要是下载依赖和构建前端），耐心等待。

脚本会自动完成：
- ✅ 检查并安装Docker/Docker Compose
- ✅ 创建必要的数据目录
- ✅ 检查.env.prod配置
- ✅ 构建Docker镜像
- ✅ 启动服务

## 六、验证服务

### 1. 检查容器运行状态
```bash
# 在/opt/shushui-ai目录下
docker compose ps
```

应该看到两个容器都是 `Up` 状态：
- shushui-backend
- shushui-frontend

### 2. 查看服务日志
```bash
# 查看所有日志
docker compose logs -f

# 只看后端日志
docker compose logs -f backend

# 只看前端Nginx日志
docker compose logs -f frontend
```

按 `Ctrl+C` 退出日志查看。

### 3. 浏览器访问
在浏览器中打开：
- **首页**：`http://你的服务器IP`
- **API文档**：`http://你的服务器IP/docs`
- **健康检查**：`http://你的服务器IP/health`

看到页面正常加载即部署成功🎉

## 八、域名与HTTPS配置（可选）

如果需要使用域名访问（更正式），需要：

### 1. 购买域名
- 阿里云/腾讯云/火山引擎都可以购买，.com域名约50-70元/年
- 推荐在火山引擎购买，方便后续备案

### 2. 域名备案
国内服务器必须备案才能使用域名访问，备案通过火山引擎控制台提交，通常需要7-15个工作日。

### 3. 配置域名解析
在域名控制台添加A记录，指向服务器公网IP。

### 4. 配置Nginx HTTPS（使用Let's Encrypt免费证书）
需要我后续帮你配置的话随时说。

## 九、数据备份

项目核心数据在以下目录，请定期备份：

```
/opt/shushui-ai/backend/
├── uploads/          # 上传的文档（最重要！）
├── vector_db/        # 向量数据库
├── shushui_ai.db     # SQLite数据库（用户信息、订单等）
```

### 备份命令示例
```bash
# 打包备份
cd /opt

# 下载到本地（在本地电脑执行）
scp root@你的服务器IP:/opt/shushui-backup-*.tar.gz ~/Downloads/
```

## 十、常见问题排查

### ❌ 浏览器访问超时/无法连接
1. 检查安全组是否开放了80端口
2. 检查容器是否正常运行：`docker compose ps`
3. 检查服务器防火墙：`ufw status`（如果开了ufw需要开放80端口）

### ❌ 页面加载出来但API请求失败
1. 检查 `.env.prod` 中 `SERVER_PUBLIC_URL` 是否正确
2. 检查后端日志：`docker compose logs backend`
3. 直接访问 `http://你的IP/health` 看后端是否正常

### ❌ AI问答没有响应/卡住
1. 检查火山方舟API Key是否正确
2. 确认 `MOCK_MODE=false`
3. 查看后端日志是否有LLM调用错误
4. 确认服务器可以访问外网（调用火山方舟API需要）：
   ```bash
   curl https://ark.cn-beijing.volces.com/api/v3
   ```

### ❌ 文件上传失败
1. 检查Nginx配置的 `client_max_body_size`（已设置100M）
2. 检查后端容器内 `uploads` 目录权限
3. 查看后端日志中的错误信息

### ❌ 服务器重启后服务没自动启动
Docker已配置 `restart: always`，正常情况下会自动启动。如果没有：
```bash
cd /opt/shushui-ai
docker compose up -d
```

### 性能优化建议
- 如果多人同时使用感觉卡，考虑升级到4核8G或更高配置
- 向量化模型（text2vec-base-chinese）首次加载需要约1-2分钟内存占用约1-2GB
- 后续用户量大了，可以考虑把SQLite换成PostgreSQL
